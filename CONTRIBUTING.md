# Contributing to Agentic-LXD-Framework

Thank you for your interest in contributing to the Agentic-LXD-Framework. This project aims to bridge the gap between Agentic AI systems and Learning Experience Design (LXD) with high structural precision.

To maintain academic integrity and professional standards, please adhere to the following guidelines.


## Academic Writing Standards
All contributions, including documentation and code comments, must follow these rules:
* **No Contractions**: Use formal English. For example, use "do not" instead of "don't" and "cannot" instead of "can't".
* **Active Voice**: Prefer active voice for clarity (e.g., "The agent processes data..." instead of "Data is processed by the agent...").
* **Precise Terminology**: Utilize academic terms such as *Pedagogical Alignment*, *RAG Optimization*, and *Metacognitive Scaffolding*.

## Educational Framework
When contributing to instructional templates or RPP structures:
* **Bloom's Taxonomy**: All learning objectives must use measurable action verbs (e.g., *Analyze, Synthesize, Evaluate*).
* **SMART Goals**: Ensure all educational outcomes are Specific, Measurable, Achievable, Relevant, and Time-bound.

## Technical Contributions
* **RAG-Optimized Data**: Ensure any data structures provided are optimized for Retrieval-Augmented Generation.
* **Documentation**: Update the README if your changes introduce new features or workflows.

## How to Submit
1. **Open an Issue**: Discuss the proposed change before starting work.
2. **Fork and Branch**: Create a feature branch from the `main` branch.
3. **Pull Request (PR)**: Submit your PR with a clear, professional description of the changes.
